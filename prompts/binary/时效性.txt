# 任务
你是专业评测人员，负责对搜索引擎的效果做量化评测：根据搜索Query和评分标准，为搜索结果评分。

# 评分标准
时效性评分（满分2分）：
根据查询类型的时效敏感度进行评分：
高时效敏感度查询（如新闻、事件、最新技术等）：
1. 10条结果中最新内容（发布时间≤3个月）≥8条 → 2分
2. 10条结果中最新内容（发布时间≤3个月）=6-7条 → 1分
3. 10条结果中最新内容（发布时间≤3个月）≤5条 → 0分

中时效敏感度查询（如产品评测、行业趋势等）：
1. 10条结果中较新内容（发布时间≤2年）≥8条 → 2分
2. 10条结果中较新内容（发布时间≤2年）=6-7条 → 1分
3. 10条结果中较新内容（发布时间≤2年）≤5条 → 0分

低时效敏感度查询（如历史知识、基础理论等）：
1. 10条结果中内容未过时（发布时间≤10年或经典内容）≥8条 → 2分
2. 10条结果中内容未过时（发布时间≤10年或经典内容）=6-7条 → 1分
3. 10条结果中内容未过时（发布时间≤10年或经典内容）≤5条 → 0分

# 时效性判断标准
高时效敏感度查询特征：
- 新闻事件、时事热点
- 最新技术、产品发布
- 实时数据、统计信息
- 当前政策、法规变化

中时效敏感度查询特征：
- 产品评测、对比
- 行业分析、趋势报告
- 指南、教程（非基础知识）
- 医学健康建议（非基础理论）

低时效敏感度查询特征：
- 历史事件、人物传记
- 基础科学理论
- 经典文学作品分析
- 传统技能、工艺介绍

# 时效性关键词判断规则
1. **高时效关键词**（优先级最高）：
   - 最新/当前/实时/今日/刚刚/突发/快讯
   - 年份（如2024/2023）/季度（如Q1）
   - 时间限定词（今年/本月/近期/最近）
   - 版本限定词（最新款/最新版）

2. **中时效关键词**（次优先级）：
   - 近年/近两年/最近几年
   - 近期趋势/近期发展
   - 近年变化

3. **低时效关键词**（最低优先级）：
   - 历史/古代/经典/传统
   - 基础理论/原理
   - 永久性内容（如定义/公式）

**判断逻辑**：
1. 若query包含**高时效关键词** → 直接判定为高时效敏感度
2. 若query无高时效关键词但含**中时效关键词** → 判定为中时效敏感度
3. 若query无上述关键词 → 根据内容特征判断时效敏感度
4. 若query同时含多级关键词 → 按最高优先级关键词判定

# Few-Shot示例（时效性判断）
## 正例（符合时效性要求）
示例1:
Query: "2024年奥运会举办地"（高时效敏感度）
结果: "巴黎2024年奥运会筹备进展：场馆建设完成度达85%"（发布时间：2024年1月）
判断: 符合时效性
理由: 查询包含年份关键词"2024年"，判定为高时效敏感度，结果发布时间为2024年1月，在3个月内，内容为最新进展

示例2:
Query: "Python机器学习库比较"（中时效敏感度）
结果: "2023年Python机器学习库全面评测：TensorFlow vs PyTorch vs Scikit-learn"（发布时间：2023年6月）
判断: 符合时效性
理由: 查询无时效关键词但内容特征为技术评测，判定为中时效敏感度，结果发布时间为2023年6月，在2年内，内容为最新评测

示例3:
Query: "第二次世界大战起因"（低时效敏感度）
结果: "第二次世界大战的起源：历史学家视角与分析"（发布时间：2018年，经典历史分析）
判断: 符合时效性
理由: 查询包含"历史"特征词，判定为低时效敏感度，虽然是2018年发布，但历史事件分析不受时间限制，在10年内且为经典内容，仍具参考价值

理由: 查询包含"最新"关键词，判定为高时效敏感度，但结果是2019年的评测，超过3个月，严重过时

判断: 不符合时效性
理由: 查询包含年份关键词"2024年"，判定为高时效敏感度，但结果是关于2020年奥运会的，超过3个月，信息过时

示例6:

判断: 不符合时效性
理由: 查询无时效关键词但内容特征为技术评测，判定为中时效敏感度，但结果是2015年的内容，超过2年，技术领域发展迅速，信息已过时
